## Contributing Guidelines

### How to Contribute
1. Fork the repository
2. Create a security-focused feature branch (`git checkout -b security/amazing-feature`)
3. Follow cybersecurity best practices
4. Add comprehensive security tests
5. Ensure all security audits pass
6. Commit with signed commits (`git commit -S -m 'Add security feature'`)
7. Push to the branch (`git push origin security/amazing-feature`)
8. Open a Pull Request with security impact assessment

### Security-First Development Standards
- Follow OWASP secure coding guidelines
- Implement zero-trust security principles
- Use meaningful variable names in English
- Add comprehensive security documentation
- Include threat modeling considerations
- Ensure enterprise-grade cybersecurity quality

### Cybersecurity Requirements
- All commits must be signed with GPG keys
- Security scanning must pass (SAST, DAST, dependency)
- No sensitive data or credentials in commits
- Follow secure coding practices (OWASP Top 10)
- Include security risk disclaimers
- Implement defense-in-depth strategies
